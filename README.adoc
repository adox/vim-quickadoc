== Overview
A minimal Neovim plugin for creating timestamped AsciiDoc notes and searching them quickly via FZF or Telescope.


== Features
* Create new `.adoc` notes with a single keymap (`<leader>en` by default)
* Notes are automatically named like `log-YYYYMMDD-HHMMSS.adoc`
* Adds headers with current timestamp and revision date
* Jump straight to the end of newly created notes
* FZF/Telescope integration for searching notes (`<leader>fn`)
* Works with either *fzf.vim* (`:Rg`) or *Telescope*


== Installation


=== Lazy.nvim

[,lua]
----
  {
    "adox/vim-quickadoc",
    config = function()
      require("quickadoc").setup({
        dir = "~/notes",          -- where to create files
        header_template = "= Log {timestamp}\n:revdate: {date}\n\n",
      })
    end,
  },
----


== Default Keymaps
[cols="1,2", options="header"]
|===
| Key | Action

| `<leader>en` 
| Create new AsciiDoc note

| `<leader>fn` 
| Search inside notes (FZF/Telescope)
|===


== Commands
* `:AdocLog` – Create a new timestamped AsciiDoc note.


== Configuration Options
[cols="1,1,3", options="header"]
|===
| Option 
| Default 
| Description

| `dir` 
| `~/github/notes` 
| Target directory for notes

| `enable_default_mapping` 
| `true` 
| Whether to install default keymaps

| `create_map` 
| `<leader>en` 
| Mapping for creating new notes

| `find_map` 
| `<leader>fn` 
| Mapping for searching notes

| `header_template` 
| `= Log {timestamp}\n:revdate: {date}\n\n` 
| Header text written to each note

| `open_cmd` 
| `edit` 
| How to open new notes (edit/tabedit/vnew/split)

| `ripgrep_args` 
| `{ "--hidden", "--glob", "!**/.git/*" }` 
| Extra arguments passed to ripgrep
|===

== Example Workflow
1. Press `<leader>en` to create a new note.
2. Start typing your log or idea.
3. Press `<leader>fn` to fuzzy search across all notes.


== Requirements
* Neovim ≥ 0.8
* `fzf.vim` or `telescope.nvim`
* `ripgrep` installed and on `$PATH`
